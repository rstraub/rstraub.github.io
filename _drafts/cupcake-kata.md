Code and Sweets. Sounds great doesn't it? Let's see what a code kata combining them can teach us.

## Code Kata With a Topping of Design Patterns 

Any form of mastery demands practice, and this code kata can help master a few specific skills too.

The Cupcake Kata is aimed at improving your proficiency at Design Patterns (though it could be solved without them). What sets it apart is that is does so by offering a fun little puzzle  as the context for applying them.

Curious what the puzzle is about? Read on.

## Sweet. Cupcakes

The premise of this exercise is twofold.

Firstly you're tasked to write code that can produce *names* and *prices* for different types of confectionaries, namely cupcakes and cookies. 

But there's a twist: different sorts of toppings can be sprinkled on top of the confectionary. To "top it off" the order in which they are added to the cupcake or cookie matters (i.e. nuts on a chocolate cupcake is different from chocolate on a nut cupcake). The name, and price vary according to the confectionary and toppings it has.

Secondly your program should allow for bundles of confectionaries. 

For instance an order can contain a bundle of two chocolate cupcakes, and a bundle of three cookies. These bundles can also be asked for their name, and total price.

How would you solve this problem?
